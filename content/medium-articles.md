## Tables Aren't the Problem. Your Mental Model of Tables Is.

**Published:** February 7, 2026 | **Updated:** February 11, 2026
**Categories:** UX, Product Design, UX Design, UI, Design Systems
**URL:** [Read on Medium](https://medium.com/@chetank19/designing-tables-in-complex-logistics-products-fa003ebaca56)

### Summary

Examines table design in complex logistics products. Argues that the real design problem with tables in complex products (logistics, finance, healthcare) isn't how the table looks — it's what the table thinks it is.

### Key Points

- Documents a complete redesign of data tables across six logistics products
- Diagnosis of table failures in B2B products:
  - Data consolidation issues
  - Lack of visual hierarchy
  - Database schema mirroring in UI
- User research across three enterprise clients revealing conflicting data importance ratings
- Five-part design framework:
  1. Read-only vs. data-entry tables
  2. Double-row cells
  3. Column zones
  4. Data consolidation
  5. Customizable views
- Implementation results: improved onboarding and reduced support tickets
- Core philosophy: effective tables require answering "what is this table for?" before design begins

This article directly connects to the Journey Management Redesign project and demonstrates system-level thinking about data display — a core portfolio theme.
